Sentry API Documentation

Welcome to the Sentry API documentation. This guide will help you integrate Sentry's Pi Network payment and blockchain-verified review solutions into your applications.

Looking for a quick integration?

Check out our Quick Start Guide to get up and running in minutes.

Overview

The Sentry API is organized around REST principles. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

API Endpoints

All API endpoints are prefixed with https://api.sentry.com/v1/

Request/Response Format

The API accepts and returns JSON format for all endpoints. Make sure to set the Content-Type: application/json header in your requests.

{
  "success": true,
  "data": {
    "transaction_id": "tx_12345",
    "amount": 100,
    "currency": "PI",
    "status": "completed"
  }
}

Quick Start Guide

Get started with Sentry in minutes by following these simple steps:

Step 1: Create a Sentry Account

Sign up for a Sentry account at dashboard.sentry.com/signup.

Step 2: Generate API Keys

Navigate to the API Keys section in your dashboard and create a new API key pair. Keep your secret key secure and never expose it in client-side code.

Step 3: Install the SDK

Install our JavaScript SDK using npm:

npm install @sentry/payments

Step 4: Initialize the SDK

Add the following code to your application:

import { Sentry } from '@sentry/payments';

Sentry.initialize({
  publicKey: 'your_public_key',
  environment: 'production'
});

Step 5: Create Your First Payment

Implement a simple payment form in your application:

const paymentButton = document.getElementById('pay-button');
paymentButton.addEventListener('click', async () => {
  try {
    const payment = await Sentry.createPayment({
      amount: 100,
      currency: 'PI',
      description: 'Product purchase',
      metadata: {
        customer_id: 'cus_12345',
        order_id: 'ord_6789'
      }
    });

    console.log('Payment created:', payment);

    // Redirect to payment page
    window.location.href = payment.checkout_url;
  } catch (error) {
    console.error('Payment error:', error);
  }
});

Authentication

Sentry uses API keys to authenticate requests. You can view and manage your API keys in the Sentry Dashboard.

API Keys

There are two types of API keys:

  • Public Key: Safe to include in client-side code. Used for initializing the SDK and creating payment sessions.
  • Secret Key: Must be kept secure and only used in server-side code. Used for API calls that require higher privileges.

Authentication Methods

There are two ways to authenticate with the Sentry API:

1. HTTP Header (Recommended)

Authorization: Bearer sk_test_12345

2. Query Parameter

https://api.sentry.com/v1/payments?api_key=sk_test_12345

Security Warning

Never share your secret API key in publicly accessible areas such as GitHub, client-side code, or in your public-facing application files.

Environments

Sentry provides two environments for development and production use:

Test Environment

The test environment is for development and testing. It uses simulated Pi Network transactions that don't involve real currency. Use test API keys (prefixed with pk_test_ and sk_test_) for this environment.

Sentry.initialize({
  publicKey: 'pk_test_12345',
  environment: 'test'
});

Production Environment

The production environment processes real Pi Network transactions. Use production API keys (prefixed with pk_live_ and sk_live_) for this environment.

Sentry.initialize({
  publicKey: 'pk_live_12345',
  environment: 'production'
});

Tip

Always test your integration thoroughly in the test environment before switching to production.

Pi Network Payments Overview

Sentry's payment solution allows you to accept Pi cryptocurrency payments in your application. Our system handles the complexities of blockchain transactions, providing a simple, reliable payment experience.

Key Features

  • Seamless Pi Network payment integration
  • Customizable checkout experience
  • Detailed transaction history and reporting
  • Webhook notifications for payment events
  • Fraud protection and security measures

Payment Flow

  1. Your application creates a payment intent using our API or SDK
  2. The customer is redirected to our secure checkout page or your custom UI
  3. The customer approves the Pi payment in their Pi wallet
  4. Sentry verifies the transaction on the Pi blockchain
  5. Your application receives a webhook notification or can poll for the payment status
