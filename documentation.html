Sentry API Documentation

Welcome to the Sentry API documentation. This guide will help you integrate Sentry's Pi Network payment and blockchain-verified review solutions into your applications.

Looking for a quick integration?

Check out our Quick Start Guide to get up and running in minutes.

Overview

The Sentry API is organized around REST principles. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

API Endpoints

All API endpoints are prefixed with https://api.sentry.com/v1/

Request/Response Format

The API accepts and returns JSON format for all endpoints. Make sure to set the Content-Type: application/json header in your requests.

{
  "success": true,
  "data": {
    "transaction_id": "tx_12345",
    "amount": 100,
    "currency": "PI",
    "status": "completed"
  }
}

Quick Start Guide

Get started with Sentry in minutes by following these simple steps:

Step 1: Create a Sentry Account

Sign up for a Sentry account at dashboard.sentry.com/signup.

Step 2: Generate API Keys

Navigate to the API Keys section in your dashboard and create a new API key pair. Keep your secret key secure and never expose it in client-side code.

Step 3: Install the SDK

Install our JavaScript SDK using npm:

npm install @sentry/payments

Step 4: Initialize the SDK

Add the following code to your application:

import { Sentry } from '@sentry/payments';

Sentry.initialize({
  publicKey: 'your_public_key',
  environment: 'production'
});

Step 5: Create Your First Payment

Implement a simple payment form in your application:

const paymentButton = document.getElementById('pay-button');
paymentButton.addEventListener('click', async () => {
  try {
    const payment = await Sentry.createPayment({
      amount: 100,
      currency: 'PI',
      description: 'Product purchase',
      metadata: {
        customer_id: 'cus_12345',
        order_id: 'ord_6789'
      }
    });

    console.log('Payment created:', payment);

    // Redirect to payment page
    window.location.href = payment.checkout_url;
  } catch (error) {
    console.error('Payment error:', error);
  }
});

Authentication

Sentry authenticates your API requests using your account's API keys. If you do not include your key when making an API request, or use an incorrect or outdated one, Sentry returns a 401 - Unauthorized HTTP response code.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail. Remember to handle API errors gracefully in your application, including authentication failures.

API Keys

Your API keys are available in the Sentry Dashboard under Developer Settings. There are two types of keys, each serving a distinct purpose:

  • Public Key (pk_...): This key is intended for use in client-side code, such as Sentry.js or mobile applications. It can be safely published as it only has limited permissions, primarily to create payment intents and tokenize information.
  • Secret Key (sk_...): This key must be kept confidential and stored securely on your own servers. Your secret API key can perform any API request to Sentry without restriction, including creating charges, issuing refunds, or managing data. Never expose your secret key in client-side code.

Using API Keys

To authenticate your server-side API requests, provide your secret API key in the Authorization header of your HTTP requests, using Bearer authentication. For client-side SDKs, you'll typically initialize the library with your public key.

Server-Side (Bearer Token):

curl https://api.sentrypayments.com/v1/payments \
  -H "Authorization: Bearer sk_test_YOUR_SECRET_KEY"

Client-Side (JavaScript SDK Example):

// Initialize Sentry.js with your public key
const sentry = Sentry('pk_test_YOUR_PUBLIC_KEY');
// The SDK will then handle including this key in relevant client-side requests.

Security Best Practice

Never share your secret API key (sk_...) in publicly accessible areas such as GitHub, client-side code, or in your public-facing application files. Treat it with the same level of confidentiality as your passwords.

Environments

Sentry provides two distinct environments to ensure a safe and robust development lifecycle: Test mode and Live mode. Each mode has its own independent set of API keys and data. There is no data sharing between the two environments.

Test Environment

The Test environment is designed for development, integration, and testing purposes. It uses simulated Pi Network transactions that do not involve real Pi currency, allowing you to build and verify your integration without any financial risk. Use test API keys (prefixed with pk_test_ and sk_test_) for all interactions with this environment.

Test API Endpoint: https://api.sentrypayments.com/v1/test/... (Note: Actual endpoints might not always include '/test/', refer to specific API calls for correct paths. The key itself dictates the environment.)

// Example: Initialize Sentry.js for Test environment
Sentry.initialize({
  publicKey: 'pk_test_YOUR_PUBLIC_KEY', // Your actual test public key
});
// Our SDKs and API typically infer the environment from the key prefix.

Live (Production) Environment

The Live environment is used for processing real Pi Network transactions with actual Pi currency. Once your integration is thoroughly tested and you are ready to start accepting real payments, you can switch to Live mode using your production API keys (prefixed with pk_live_ and sk_live_).

Live API Endpoint: https://api.sentrypayments.com/v1/...

// Example: Initialize Sentry.js for Live environment
Sentry.initialize({
  publicKey: 'pk_live_YOUR_PUBLIC_KEY', // Your actual live public key
});

Important Note on Data

Objects created in one mode (e.g., a payment link in Test mode) are not accessible or visible in the other mode. Always ensure you are using the correct set of API keys corresponding to the environment you intend to interact with.

Pi Network Payments Overview

Sentry's Pi Network Payments solution empowers your business to seamlessly accept Pi cryptocurrency from a global user base. Our platform is engineered to abstract the complexities of blockchain interactions, delivering a straightforward, secure, and reliable payment experience for both merchants and their customers.

Key Features

  • Effortless Integration: Quickly embed Pi payment capabilities into your website, mobile app, or platform using our versatile SDKs or by integrating directly with our well-documented REST API.
  • Secure & Customizable Checkout: Utilize Sentry's secure, hosted checkout page for a quick setup, or build a fully custom payment flow tailored to your brand's user experience.
  • Real-time Transaction Confirmations: Benefit from the inherent speed of the Pi blockchain, coupled with Sentry's optimized processing, for swift transaction confirmations.
  • Comprehensive Merchant Dashboard: Monitor all your Pi transactions, manage payments and refunds, and access detailed financial reporting directly from your Sentry merchant dashboard.
  • Automated Webhook Notifications: Stay informed with real-time, automated updates for crucial payment events such as successful transactions, processing failures, or refund status changes.
  • Advanced Security & Fraud Prevention: Operate with confidence knowing Sentry employs robust security measures and intelligent fraud detection systems specifically designed for Pi transactions.

Typical Payment Flow Explained

  1. Payment Initiation: Your application creates a payment intent or a shareable payment link via the Sentry API/SDK, specifying the transaction amount in Pi, order details, and callback URLs.
  2. Customer Authorization: The customer is directed to a secure Sentry-hosted page (or your custom UI) where they authenticate with their Pi wallet and explicitly approve the transaction details.
  3. Blockchain Interaction & Verification: Sentry securely processes the transaction, submits it to the Pi blockchain for validation by the network, and diligently awaits confirmation.
  4. Status Confirmation: Once the transaction achieves finality on the Pi blockchain, Sentry updates the payment status accordingly within our system.
  5. Real-time Notification: Your application is promptly notified of the successful payment (or any failure) via a configured webhook, or it can poll the payment status using the API for an update.
  6. Order Fulfillment: Upon confirmation of a successful payment, you can confidently fulfill the customer's order or provide the agreed-upon service.

Webhooks for Real-Time Event Notifications

Sentry utilizes webhooks to proactively notify your application about events that occur in your Sentry account. Webhooks are essential for handling asynchronous events efficiently, such as when a customer's Pi payment is successfully confirmed, a new blockchain-verified review is submitted, or a payment dispute is initiated.

By subscribing to webhook events, your application can react immediately to these occurrences without the need for continuous API polling, leading to more responsive and efficient system interactions. This is the recommended way to get notified of changes to the status of your Sentry objects.

Configuring Your Webhook Endpoints

You can add, manage, and test your webhook endpoints directly from the "Developer Settings" -> "Webhooks" section of your Sentry Merchant Dashboard. To add an endpoint, you'll need to provide a publicly accessible HTTPS URL that Sentry can send HTTP POST requests to. You can also select the specific event types you want to subscribe to for each endpoint.

Securing Your Webhook Endpoints

Ensuring the security of your webhook endpoints is critical. You must verify that incoming webhook requests are genuinely from Sentry and not from a malicious third party. Sentry signs each webhook event it sends by including a unique signature in the Sentry-Signature HTTP header. This signature is generated using a secret key specific to your webhook endpoint, which you can find in your Sentry Dashboard. Verifying this signature on your server confirms the authenticity and integrity of the payload. Detailed instructions on signature verification are available in the "Security" section of our API documentation.

Supported Event Types (Examples)

  • payment.succeeded: Sent when a Pi payment is successfully processed and confirmed.
  • payment.failed: Sent when a Pi payment attempt fails.
  • review.created: Sent when a new blockchain-verified review is submitted.
  • payout.created: Sent when a payout to your account is initiated.
  • (More event types would be listed here...)

Refer to the Webhook Event Types API reference for a complete list of events and their payload structures.

Blockchain Reviews API

The Sentry Review API provides programmatic access to manage and leverage your blockchain-verified customer reviews. This API enables you to integrate the trust and transparency of Sentry reviews directly into your applications, websites, and backend systems.

Common Use Cases & Benefits

  • Display Authentic Reviews: Fetch and showcase verified reviews on your product pages, testimonials sections, or marketing materials to build customer confidence.
  • Automate Review Requests: Integrate review submission prompts into your post-purchase customer journeys (e.g., via email or in-app notifications) to encourage feedback.
  • Data Analysis & Insights: Aggregate review data, analyze sentiment, and identify trends to gain deeper insights into customer satisfaction and product performance.
  • System Synchronization: Keep review data consistent across your various business systems, such as CRM or customer support platforms.

Key Endpoints & Functionality

  • GET /v1/reviews: Retrieve a list of verified reviews, with options for filtering by product, rating, date, etc.
  • GET /v1/reviews/{review_id}: Fetch a specific review by its unique ID, including its verification details on the Pi blockchain.
  • POST /v1/reviews/request: Programmatically initiate a review request to a customer after a confirmed transaction.
  • GET /v1/products/{product_id}/stats: Retrieve review statistics for a specific product, such as average rating and review count.

Example: Retrieving verified reviews for a specific product with a minimum rating:

curl https://api.sentrypayments.com/v1/reviews \
  -H "Authorization: Bearer sk_test_YOUR_SECRET_KEY" \
  -G \
  --data-urlencode "product_id=prod_A4B8C2" \
  --data-urlencode "min_rating=4" \
  --data-urlencode "limit=10"

By leveraging the Review API, you can fully integrate the power of Sentry's blockchain-verified reviews, enhancing transparency and building stronger trust with your user base.
